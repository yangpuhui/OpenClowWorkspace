## 2. 学习输出规范
### 2.1 文档格式要求
- **命名规则**: Java学习-{模块}-{知识点}.md（例：Java学习-编码规范-命名规范.md）
- **核心结构**:
 1. 核心概念（通俗解释+生活案例）
 2. 代码Demo（≤50行，完整注释，符合阿里规范）
 3. 梯度练习题（2-3道，基础→进阶→实战）
 4. 常见坑点（错误案例+解决方案）
- **格式适配**: Markdown格式，兼容Obsidian双向链接，支持知识点关联

### 2.2 代码风格要求
- **代码规模**: 单Demo代码行数≤50行，拆分为小模块讲解
- **注释规范**: 类/方法/关键变量需加Javadoc注释，符合阿里Java开发手册
- **可运行性**: 代码可直接复制到IntelliJ IDEA运行，无需额外配置
- **规范遵循**: 严格遵守命名、异常处理、集合使用等阿里手册规则

## 3. 沟通交互偏好
- **讲解逻辑**: 从已有基础出发，不重复入门知识；按「是什么→怎么用→为什么」步骤讲解
- **概念解释**: 抽象知识点需结合生活案例（如用"快递驿站"解释Redis缓存）
- **呈现形式**: 优先使用列表/表格梳理内容，避免大段文字
- **术语使用**: 专业术语需先解释概念（如"幂等性"需先解释概念再讲解）

## 4. 工具适配要求

- **代码运行**: 适配IntelliJ IDEA开发环境
- **进度记录**: 自动同步学习进度到MEMORY.md，标记已完成/待学习知识点
