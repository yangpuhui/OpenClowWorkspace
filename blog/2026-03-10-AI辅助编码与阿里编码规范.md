**是什么**：Java 21引入的轻量级线程，让并发编程像写普通代码一样简单。

## 二、Java后端学习查漏补缺/进阶

### 核心知识点：集合使用的性能陷阱

你刚学习了"命名规范"和"常量与变量"，接下来要学"集合使用"。阿里规范中，集合是**踩坑最多的地方**。

#### 1. 核心概念

**通俗解释**：
- **ArrayList**：像公寓楼，每个房间住一个人，按门牌号找很快（随机访问O(1)），但插入删除要移动其他人（O(n)）
- **LinkedList**：像火车车厢，每节车厢前后相连，插入删除只要解开挂钩就行（O(1)），但找人要从头车厢走到尾车厢（随机访问O(n)）
- **HashSet**：像快递柜，每个柜子有编号，放东西和找东西都很快（O(1)），但无序

**生活案例（宠物医疗预约）**：
- **ArrayList**：预约列表按时间排序，适合遍历（显示今日所有预约）
- **HashSet**：快速判断某宠物是否已预约（避免重复预约）
- **HashMap**：宠物ID → 预约记录，适合查询（根据ID查预约详情）

#### 2. 极简代码Demo（≤50行，符合阿里规范）

```java
package com.pet.clinic.appointment;

import java.util.*;

/**
 * 预约集合使用示例
 * 演示ArrayList、HashSet、HashMap的正确使用
 *
 * @author yph
 * @since 2026-03-10
 */
public class AppointmentCollectionDemo {

    /** 预约列表（适合遍历显示） */
    private final List<Appointment> appointmentList = new ArrayList<>();

    /** 已预约宠物ID集合（适合去重） */
    private final Set<Long> bookedPetIds = new HashSet<>();

    /** 宠物ID→预约记录映射（适合快速查询） */
    private final Map<Long, Appointment> appointmentMap = new HashMap<>();

    /**
     * 创建预约
     *
     * @param petId 宠物ID
     * @param appointment 预约信息
     * @return true创建成功，false宠物已预约
     */
    public boolean createAppointment(Long petId, Appointment appointment) {
        if (bookedPetIds.contains(petId)) {
            return false;  // 避免重复预约
        }

        bookedPetIds.add(petId);
        appointmentList.add(appointment);
        appointmentMap.put(petId, appointment);
        return true;
    }
}
```

#### 3. 常见坑点

**❌ 坑点1：频繁扩容**
```java
// 错误示例
List<Appointment> list = new ArrayList<>();  // 默认容量10
for (int i = 0; i < 1000; i++) {
    list.add(appointment);  // 10→15→22→33→... 多次扩容，性能差
}

// ✅ 正确做法：预估大小
List<Appointment> list = new ArrayList<>(1000);  // 一次性分配，避免扩容
```

**❌ 坑点2：遍历时删除元素**
```java
// 错误示例
for (Appointment apt : list) {
    if (apt.isExpired()) {
        list.remove(apt);  // ❌ ConcurrentModificationException
    }
}

// ✅ 正确做法：使用Iterator
Iterator<Appointment> it = list.iterator();
while (it.hasNext()) {
    if (it.next().isExpired()) {
        it.remove();  // ✅ 安全删除
    }
}
```

**❌ 坑点3：HashSet未重写equals/hashCode**
```java
// 错误示例
class Pet {
    private Long id;
    // 没重写equals和hashCode，HashSet无法正确判断重复
}

// ✅ 正确做法：重写equals和hashCode
class Pet {
    private Long id;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Pet)) return false;
        Pet pet = (Pet) o;
        return Objects.equals(id, pet.id);
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);
    }
}
```
