## 二、Java后端学习查漏补缺/进阶

### 核心知识点：ArrayList vs HashMap vs HashSet 选择指南

你现在在学"集合使用"，但很多同学只知道"HashMap存键值对、List存列表"，不知道该什么时候用什么。今天帮你彻底搞懂！

#### 1. 核心概念（生活类比）

**三种集合的生活场景：**

| 集合类型 | 生活类比 | 特点 | 时间复杂度 |
|---------|---------|------|-----------|
| **ArrayList** | 排队的人 | 按顺序存储，可重复，索引查找快 | get: O(1), add: O(1), contains: O(n) |
| **HashSet** | 学生花名册 | 无序，不重复，快速判断是否存在 | add: O(1), contains: O(1) |
| **HashMap** | 快递取件柜 | 键值对存储（柜号→包裹），快速查找 | put/get: O(1) |

**口诀记忆：**
- 要顺序选 **ArrayList**（比如预约列表按时间排）
- 要去重选 **HashSet**（比如已预约的宠物ID不重复）
- 要映射选 **HashMap**（比如ID→预约详情）

#### 2. 代码Demo（≤50行，符合阿里规范）

```java
package com.pet.clinic.appointment;

import java.util.*;

/**
 * 集合使用示例 - 宠物医疗预约平台
 * 对比ArrayList/HashSet/HashMap的使用场景
 *
 * @author yph
 * @since 2026-03-12
 */
public class CollectionDemo {

    // 场景1：存储预约列表 - 需要按时间顺序
    private final List<String> appointmentList = new ArrayList<>(100);

    // 场景2：检查重复预约 - 快速判断是否已预约
    private final Set<Long> bookedPetIds = new HashSet<>(100);

    // 场景3：查询预约详情 - 通过ID快速获取
    private final Map<Long, String> appointmentMap = new HashMap<>(100);

    public void demo() {
        // 添加预约
        appointmentList.add("2026-03-12 09:00");
        bookedPetIds.add(1001L);
        appointmentMap.put(1001L, "2026-03-12 09:00 张医生");

        // 查询（O(1)时间复杂度）
        boolean isBooked = bookedPetIds.contains(1001L);  // true
        String detail = appointmentMap.get(1001L);  // "2026-03-12 09:00 张医生"
    }
}
```

#### 3. 常见坑点（错误案例 + 解决方案）

**❌ 坑点1：用ArrayList判断存在，效率低**

```java
// ? 错误示例 - contains是O(n)，10000次就是100万次操作
List<Long> petIds = new ArrayList<>();
for (int i = 0; i < 10000; i++) {
    if (petIds.contains(1001L)) {  // 每次都要遍历整个列表
        // ...
    }
}

// ? 正确做法 - HashSet的contains是O(1)
Set<Long> petIds = new HashSet<>();
if (petIds.contains(1001L)) {  // 一次哈希计算搞定
    // ...
}
```

**阿里规范要求**：频繁判断元素是否存在时，使用HashSet而非ArrayList。

---

**❌ 坑点2：遍历HashMap时直接删除元素会报错**

```java
// ? 错误示例 - ConcurrentModificationException
Map<Long, String> map = new HashMap<>();
map.put(1L, "A");
map.put(2L, "B");

for (Long key : map.keySet()) {
    if (key == 1L) {
        map.remove(key);  // 报错！遍历时不能修改
    }
}

// ? 正确做法 - 使用Iterator
Iterator<Long> iterator = map.keySet().iterator();
while (iterator.hasNext()) {
    Long key = iterator.next();
    if (key == 1L) {
        iterator.remove();  // 安全删除
    }
}
```

**阿里规范要求**：遍历集合需要修改时，使用Iterator而非直接操作。

---

**❌ 坑点3：HashSet存储自定义对象未重写hashCode/equals**

```java
// ? 错误示例 - 无法正确去重
class Pet {
    private Long id;
    private String name;
    // 未重写hashCode/equals，HashSet认为每个对象都不同
}

Set<Pet> petSet = new HashSet<>();
petSet.add(new Pet(1L, "旺财"));
petSet.add(new Pet(1L, "旺财"));  // 两个对象都加入了！无法去重

// ? 正确做法 - 同时重写equals和hashCode
class Pet {
    private Long id;

    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Pet)) return false;
        Pet pet = (Pet) o;
        return Objects.equals(id, pet.id);
    }

    @Override
    public int hashCode() {
        return Objects.hash(id);  // 相同id的哈希值相同
    }
}
```

**阿里规范要求**：自定义对象作为集合元素时，必须重写equals和hashCode。

#### 4. 实战应用场景（结合宠物医疗预约平台）

**场景1：预约列表展示（ArrayList）**
```java
// 需求：按预约时间顺序展示今日所有预约
private final List<Appointment> todayAppointments = new ArrayList<>();

public List<Appointment> getTodayAppointments() {
    return new ArrayList<>(todayAppointments);  // 返回副本，避免外部修改
}
```

**场景2：防止重复预约（HashSet）**
```java
// 需求：同一宠物同一天只能预约一次
private final Set<String> dailyAppointmentKeys = new HashSet<>();

public boolean canBook(Long petId, String date) {
    String key = petId + "_" + date;
    return !dailyAppointmentKeys.contains(key);  // O(1)快速判断
}
```

**场景3：快速查询预约详情（HashMap）**
```java
// 需求：通过预约ID快速获取预约信息
private final Map<Long, Appointment> appointmentCache = new HashMap<>(1024);

public Appointment getById(Long appointmentId) {
    return appointmentCache.get(appointmentId);  // O(1)查询
}
```
